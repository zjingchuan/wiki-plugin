# /wiki-init

首次使用 wiki-plugin 时配置分类。也可重新运行修改已有配置。

## 处理流程

1. 询问用户：保留默认分类（产品/技术/运维）还是自定义？
2. 若自定义：
   - 询问每个分类的名称和描述
   - 描述会用于 Claude 后续的智能分类（越具体越准确）
3. 调用 `init_wiki_config` 写入 docs/.wiki/config.json
4. 报告：分类已生效，可以运行 /wiki-import 开始处理文档

## 默认分类

| 分类 | 描述 |
|------|------|
| 产品 | 需求、PRD、用户手册、产品介绍 |
| 技术 | 架构、接口、数据库、技术方案 |
| 运维 | 部署、监控、安全、合规 |

## 注意

- 如果 config.json 已存在，会询问是否覆盖
- 修改分类后，已存在的分类目录不会自动迁移，需用户手动整理
